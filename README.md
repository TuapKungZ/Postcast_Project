# 🌍 ReadCast Pro: Polyglot (Podcast Generator)

ระบบสร้างรายการ Podcast สนุกๆ จากบทความข่าวภาษาอังกฤษหรือเนื้อความตามสั่งโดยอัตโนมัติ ขับเคลื่อนด้วยโมเดลภาษา **Google Gemini 2.5 Flash** และการสังเคราะห์เสียงเป็นธรรมชาติผ่าน **Edge TTS** พร้อมมิกซ์ดนตรีประกอบเป็นไฟล์เสียง MP3 ทันที!

---

## ✨ ฟีเจอร์หลัก (Key Features)
1. **🤖 AI Scriptwriter:** ใช้ Gemini 2.5 Flash ในการเขียนบทสนทนาแนว Podcast โต้ตอบระหว่าง Host A และ Host B ได้อย่างลื่นไหลและเป็นธรรมชาติ
2. **🎙️ Multi-Host & Multi-Language Support:**
   - **🇹🇭 โหมดทั่วไป (สรุปข่าว):** สรุปเป็นภาษาไทย 100% (Host A เสียงผู้ชาย / Host B เสียงผู้หญิง)
   - **🇬🇧 ฝึกภาษา (English Tutor):** Host A พูดสอนภาษาอังกฤษแบบสำเนียงเจ้าของภาษา และ Host B แปล/อธิบายเป็นภาษาไทย
3. **🌐 Web Scraper:** สามารถวางลิงก์ข่าวภาษาอังกฤษ (เช่น BBC, CNN) ระบบจะไปดึงข้อมูลเนื้อหามาเขียนบทให้อัตโนมัติ
4. **🎵 Background Music Mixer:** ผสมเสียงดนตรีประกอบ (BGM) พื้นหลังเบาๆ เพิ่มความน่าฟังให้กับ Podcast (หากมีไฟล์ `music.mp3` อยู่ในโปรเจกต์)
5. **💾 Easy Download:** ผลิตเสร็จสามารถดาวน์โหลดไฟล์ `.mp3` ไปฟังได้ทันที

## 🛠️ วิธีการติดตั้งและเตรียมระบบ (Setup & Installation)

หากคุณดาวน์โหลดโค้ดนี้ไปรันในเครื่องใหม่ ให้ทำตามขั้นตอนต่อไปนี้:

1. **ติดตั้ง Python 3.9 หรือเวอร์ชันใหม่กว่า**
2. **สร้าง Virtual Environment และติดตั้งไลบรารีที่จำเป็น:**
   ```bash
   # สร้าง venv
   python -m venv venv

   # เปิดใช้งาน venv (Windows - PowerShell)
   .\venv\Scripts\Activate.ps1

   # เปิดใช้งาน venv (Windows - CMD)
   venv\Scripts\activate

   # ติดตั้ง dependencies
   pip install -r requirements.txt
   ```

3. **ตั้งค่า API Key ของ Google Gemini:**
   สร้างไฟล์ `.env` ไว้ที่โฟลเดอร์เริ่มต้นของโปรเจกต์ (Root Directory) และใส่คีย์ของตัวเองดังนี้:
   ```env
   GOOGLE_API_KEY=ใส่คีย์ของคุณที่นี่
   ```
   *(หรือสามารถกรอกโดยตรงผ่านแถบเมนูด้านซ้ายบนหน้าเว็บได้เช่นกัน)*

4. **ดนตรีประกอบ (ถ้าต้องการ):**
   นำไฟล์เพลงที่อยากให้เป็นเพลงแบ็คกราวด์ของรายการมาวางไว้ที่หน้าหลักของโปรเจกต์และตั้งชื่อว่า `music.mp3`

## 🚀 วิธีการใช้งานและรันโปรเจกต์ (Usage Instructions)

คุณสามารถเลือกใช้โปรเจกต์ได้ 2 รูปแบบตามต้องการ:

### 1. รันแบบหน้าเว็บ (Web UI ด้วย Streamlit) 🌟 *แนะนำ*
วิธีนี้ใช้งานได้สะดวก มีการแชร์คอลัมน์ซ้าย-ขวาอย่างชัดเจน สามารถแก้ไขบทพูดก่อนสั่งบันทึกเสียงจริงได้
```bash
# สำหรับ Windows
.\venv\Scripts\streamlit run app.py
```
*ระบบจะเปิดเว็บบราวเซอร์ขึ้นมาที่หน้า `http://localhost:8501` อัตโนมัติ*

### 2. รันแบบ Command Line (CLI) 💻
รันประมวลผลผ่านหน้าจอ Terminal โดยใช้บทความจากไฟล์ `source.txt` หรือวางลิงก์ข่าว
```bash
# สำหรับ Windows
.\venv\Scripts:python main.py
```
*เมื่อทำงานเสร็จแล้ว ไฟล์เสียงที่ได้จะบันทึกเป็นชื่อ `final_podcast.mp3` และระบบจะเปิดเล่นเพลงให้ทันที*
